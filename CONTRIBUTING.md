# Guide de contribution

Merci de contribuer au projet El Carino.

## Style

- JavaScript moderne
- React fonctionnel
- Hooks uniquement
- Pas de composants de classe

## Commits

Format recommandé :

```
type(scope): description
```

Exemples :

```
feat(admin): add registration list

fix(validation): correct age calculation

docs: update roadmap

style(profile): improve cards layout

refactor(form): simplify validation
```

## Types

- feat
- fix
- docs
- style
- refactor
- perf
- test
- chore

## Branches

main

branche stable

develop

développement

feature/...

nouvelle fonctionnalité

fix/...

correction

## Pull Requests

Chaque Pull Request doit :

- compiler
- passer les tests
- respecter ESLint
- être documentée
